# GM V18.1 — 多策略融合量化交易策略

基于掘金量化（GoldMiner）平台的 A 股量化交易策略，迭代至 V18.1。

| 项目 | 说明 |
|------|------|
| 策略类型 | 三策略并行 + 投票融合（MR+MOM+VP） |

## V18.1 核心架构

### 三策略并行

| 策略 | 代号 | 核心逻辑 | 市场偏好 |
|------|------|----------|----------|
| 均值回归 | MR | RSI超卖六因子打分，ATR跟踪止盈 | 震荡市（权重×3） |
| 动量趋势 | MOM | MA20>MA60多头排列+放量突破 | 牛市（权重×2） |
| 量价背离 | VP | 价新低+缩量=抛压耗尽 | 熊市（权重×2） |

### 投票融合规则

- **入场**：≥2/3 策略同意才执行，MR 始终主导（权重 1.5~3.0）
- **出场**：归属策略优先 + 止损类安全阀 + 趋势/时间交叉验证
- **仓位**：3/3 一致→满仓，2/3→70%，2 同意 1 反对→50%，分歧→40%或不交易

### MR 均值回归（主导策略）

**六因子评分体系**：

| 因子 | 权重 | 说明 |
|------|------|------|
| RSI 超卖深度 | 35% | RSI(14) < 30，越超卖分越高 |
| 趋势对齐 | 20% | 价格在 MA20 上方 |
| 成交量确认 | 15% | 放量 ≥ 20日均量 1.2 倍 |
| 近期跌幅 | 15% | 5%~20% 跌幅为最佳反弹区间 |
| 反转形态 | 8% | 看涨吞没 / 锤子线识别 |
| 行业动量 | 7% | 优选强势行业中的超卖股 |

**出场优先级**：ATR跟踪止盈(2.5x) > ATR止损(1.5x) > RSI过热(>80) > 时间止损(15天)

### MOM 动量趋势

- **入场**：MA20>MA60 多头排列 + 价格站上 MA20 + 评分≥0.50
- **出场**：跌破 MA20 / 死叉 / 止损 6% / 止盈 15% / RSI 极端 >85
- **加分**：金叉 +0.15

### VP 量价背离

- **入场**：价新低 + 缩量（量比<0.65）+ 评分≥0.50
- **出场**：放量反弹结束 / 放量滞涨 / 止损 5% / 止盈 8% / 时间止损 20 天

## 项目结构

```
├── config.py              # V18 策略参数配置（融合权重/阈值/市场状态）
├── indicators.py          # 技术指标纯函数库（RSI/ATR/MA/量比/形态）
├── stock_pool.py          # 35只精选股票池管理（12行业）
├── screener.py            # 六因子选股打分引擎（MR 策略复用）
├── strategy_mr.py         # 均值回归策略模块 (V18)
├── strategy_momentum.py   # 动量趋势策略模块 (V18)
├── strategy_vp.py         # 量价背离策略模块 (V18)
├── fusion.py              # 投票融合引擎 (V18)
├── knn_matcher.py         # KNN 历史匹配引擎 (V18)
├── main.py                # V18 策略主程序（三策略并行+融合）
├── offline_backtest.py    # V18 离线回测脚本
└── config.yaml            # 掘金终端配置
```

## 使用方法

### 1. GM 终端回测

在掘金量化终端中打开 `main.py`，点击"回测"。

### 2. 离线验证

```bash
python3 offline_backtest.py
```

不依赖掘金终端，直接用 `history()` API 拉数据模拟回测。
