# V11-V15 — 波动期：过度优化 → 删繁就简

## 版本总览

| 版本 | 策略方向 | 回测收益 | 核心思路 |
|------|----------|----------|----------|
| V11 | 分档建仓（3 档） | 回退 | RSI 触发后分 3 次建仓，意在降低成本 |
| V12 | 动量突破 + RSI | 回退 | RSI 超卖 + 放量突破确认，但参数不匹配 |
| V13 | 回归简单 RSI(6) | **+50.0%** | 删除所有复杂逻辑，只用 RSI(6)<32 + ATR 止损 |
| V14 | V13 + 移动止盈 + 60 只池 | **+56.0%** | ATR 跟踪止盈 + 扩大股票池至 60 只 |
| V15 | 行业分散 50 只 | 回退 | 缩回 50 只 + 行业分散，但信号被稀释 |

## 核心教训

1. **过度优化是最大的敌人**：V11/V12 加了太多逻辑反而回撤
2. **简单往往更有效**：V13 删繁就简后收益暴涨到 +50%
3. **移动止盈让利润奔跑**：V14 在 V13 基础上加 ATR 跟踪止盈，收益再提升 6pp
4. **行业分散不等于更好**：V15 强制行业分散导致信号稀释
5. **股票池大小有最优区间**：20 只太少、60 只最优、50 只偏少

## 策略逻辑概要

### V11：分档建仓

```
入场：RSI(6) < 32 后分 3 档建仓
  - 第一档: RSI < 32, 买入 33%
  - 第二档: RSI < 28, 再买 33%
  - 第三档: RSI < 22, 再买 34%
出场：RSI > 70 / 止损 5%
问题：分档增加了复杂度但未提升收益，反而因行情快速反弹错过最佳入场点
```

### V12：动量突破 + RSI

```
入场：RSI(6) < 35 且放量突破 MA5
出场：跌破 MA5 / 止损 4%
问题：RSI 超卖和动量突破逻辑冲突，超卖股往往在下跌趋势中，不满足突破条件
```

### V13：回归简单 RSI（历史最佳之一）

```
入场：RSI(6) < 32（简单粗暴）
出场：RSI > 70 / 止损 5%
成功原因：逻辑极简，信号清晰，无过度过滤
```

### V14：V13 + 移动止盈（历史最佳）

```
入场：同 V13
出场：ATR 跟踪止盈 2.0× / RSI > 75 / 止损 5%
      止盈改为"跟踪"而非"固定"，让盈利充分奔跑
股票池：60 只（V13 约 20 只），更多机会
```

### V15：行业分散

```
入场：同 V14
出场：同 V14
变化：60 只 → 50 只 + 行业分散约束（单一行业最多 1 只持仓）
问题：行业约束过于严格，限制了优质标的的入场机会
```

## 关键参数

| 参数 | V11 | V12 | V13 | V14 | V15 |
|------|-----|-----|-----|-----|------|
| RSI 周期 | 6 | 6 | 6 | 6 | 6 |
| RSI 买入 | 32 | 35 | 32 | 32 | 32 |
| 建仓方式 | 3档 | 单次 | 单次 | 单次 | 单次 |
| 止损 | 5% | 4% | 5% | 5% | 5% |
| 止盈 | RSI>70 | 跌破MA5 | RSI>70 | ATR×2.0 | ATR×2.0 |
| 股票池 | 50只 | 30只 | 20只 | 60只 | 50只 |
| 行业分散 | 否 | 否 | 否 | 否 | 是(1只/行业) |

## V13/V14 遗留逻辑

V13 的简单 RSI 逻辑在当前代码中仍可复现：

```python
# V13 核心入场逻辑（2 行代码）
if rsi < 32:
    buy(symbol, position_pct=0.25)

# V14 止盈改为 ATR 跟踪
trail_price = max(trail_price, close - atr * 2.0)
if close < trail_price:
    sell(symbol)
```
